# Апекс — картинг-центр

## Описание

Адаптивное клиентское веб-приложение (mobile-first) для бронирования заездов в картинг-центре «Апекс». Пользователь может просматривать расписание слотов, фильтровать их по дате, трассе и сложности, создавать бронирования, просматривать и отменять свои записи.

## Проблема заказчика и цель MVP

Владелец картинг-центра вручную вёл запись в Telegram и на доске маркером. Это приводило к ошибкам: двойная запись на один карт, потеря заказов, хаос в выходные.

**Цель MVP** — цифровой инструмент самостоятельной записи, покрывающий полный пользовательский путь от просмотра расписания до подтверждения бронирования, с обработкой ключевых сценариев (конфликты, отмена, уведомления об отмене центром).

## Технологии

- **HTML** — семантическая разметка, все экраны в одном файле
- **CSS** — mobile-first адаптивная вёрстка, БЭМ, CSS-переменные
- **JavaScript** — чистая реализация без фреймворков, ES-модули
- **Mock API** — имитация backend для учебной демонстрации
- **LocalStorage** — только для учебных mock-данных (не источник истины)
- **Git** — управление версиями

## Реализованные функции

- Каталог и фильтрация слотов (по дате, трассе, сложности)
- Подробности заезда (трасса, маршал, свободные карты, цена, адрес)
- Создание бронирования (форма с валидацией, пересчёт стоимости)
- Обработка ошибок API (400, 401, 404, 409, 422, 500, 503)
- Раздел «Мои бронирования» (список, статусы, детали)
- Отмена бронирования (с учётом `canCancel` и `cancellationDeadline`)
- Отображение отмены центром (статус, причина, без кнопки отмены)

## Инструкция запуска

1. Открыть проект в VS Code
2. Запустить через Live Server (правый клик на `index.html` → Open with Live Server)
3. Либо выполнить в корне проекта:
   ```
   python -m http.server 8080
   ```
4. Открыть в браузере: [http://localhost:8080](http://localhost:8080)

## Реальная архитектура и учебный mock-режим

**Реальная архитектура** предполагает существующий backend (black-box), который является единственным источником истины. Клиентское приложение потребляет API, обрабатывает ошибки (включая 409 Conflict), не хранит данные в LocalStorage.

**Учебный mock-режим** имитирует backend на стороне клиента через модуль `mock-api.js`. Бронирования сохраняются в LocalStorage под ключом `kartingMockBookings` — это техническое хранилище учебного mock, не являющееся источником истины. Переключение между режимами выполняется в `app.js`.
