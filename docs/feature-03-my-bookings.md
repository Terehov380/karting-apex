# Feature 03: Мои бронирования и отмена

## Цель

Предоставить клиенту возможность просматривать свои бронирования и отменять их.

## Связанные требования

- **US-06**: Просмотр списка активных и завершённых бронирований
- **US-07**: Отмена бронирования при соблюдении условий
- **FR-11**: Список бронирований клиента
- **FR-13**: Отмена бронирования с проверкой прав
- **FR-14**: Изменение статуса при отмене
- **BR-06**: Статусы booking: confirmed, cancelled_by_client, cancelled_by_center, completed

## Реализованное поведение

1. Пункт навигации «Мои бронирования» в шапке
2. Загрузка списка бронирований из LocalStorage (mock API)
3. Карточка каждого бронирования содержит: номер, дату/время, трассу, маршала, участников, экипировку, итог, статус, дату создания, срок отмены
4. Статусы визуально отличаются цветом
5. Кнопка «Отменить бронирование» — только если статус confirmed, canCancel=true и дедлайн не истёк
6. Встроенное подтверждение отмены (без alert/confirm)
7. После отмены: статус меняется, карты возвращаются в слот, список и расписание обновляются
8. Демо-бронь со статусом cancelled_by_center (не удаляется, отображается причина)
9. Empty state при отсутствии бронирований
10. Состояния загрузки и ошибки с кнопкой повтора

## Особенности mock API

- Бронирования хранятся в localStorage под ключом `kartingMockBookings`
- Только техническое хранилище учебного mock, не источник истины
- При первом запросе GET /bookings автоматически добавляется демо-бронь `bk-demo-cancelled-center` со статусом cancelled_by_center
- При отмене карты возвращаются в slotStates (участвуют в расчёте availableKarts)
- 401 не реализован (клиент не идентифицирован) — требует подтверждения механизма идентификации
