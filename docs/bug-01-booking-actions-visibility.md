# Баг 1: Видимость кнопок формы бронирования

| Поле | Значение |
|------|----------|
| **ID** | BUG-01 |
| **Название** | Кнопки «Отмена» и «Забронировать» находятся слишком низко и почти не видны на экране без дополнительной прокрутки |

## Симптом

При открытии формы бронирования на мобильных разрешениях (320–480 px) кнопки «Отмена» и «Забронировать» располагались в самом низу формы. После заполнения полей пользователю приходилось дополнительно прокручивать страницу вниз, чтобы увидеть и нажать кнопки. Это создавало риск того, что пользователь не заметит кнопку «Забронировать» и покинет форму.

## Серьёзность

**Высокая** — блокирует завершение основного пользовательского сценария (создание бронирования) на мобильных устройствах, которые являются целевой платформой.

## Окружение

- Все браузеры (Chrome, Firefox, Safari, Edge)
- Все устройства с высотой экрана менее 800 px (мобильные телефоны)
- Ширина экрана 320–480 px

## Связанные требования

- NFR-02: Адаптивная mobile-first вёрстка
- US-07: Создание бронирования
- FR-09: Экран подтверждения бронирования

## Предусловия

- Приложение загружено
- Открыт экран каталога слотов
- Выбран доступный слот
- Нажата кнопка «Записаться»

## Шаги воспроизведения

1. Открыть приложение на устройстве с высотой экрана ≤ 667 px (iPhone SE / 320 px width)
2. Выбрать доступный слот
3. Нажать «Записаться»
4. Прокрутить форму до конца

## Ожидаемый результат

Кнопки «Отмена» и «Забронировать» видны на экране без дополнительной прокрутки или фиксированы в нижней части экрана.

## Фактический результат

Кнопки находились за пределами видимой области. Пользователь вынужден прокручивать форму вручную, часть пользователей могла не заметить кнопку «Забронировать».

## Причина

- Форма бронирования не имела ограничения высоты с прокруткой содержимого
- Блок действий (`.booking-form__actions`) не был зафиксирован (position: sticky) в нижней части экрана
- Отсутствовал нижний отступ (padding-bottom), чтобы содержимое формы не перекрывалось блоком действий

## Исправление

В `css/style.css` были внесены следующие изменения:

1. **Прокручиваемый контейнер формы**: добавлен CSS для `#screen-booking-form.screen--active`:
   - `height: calc(100vh - var(--header-height))` — высота экрана минус шапка
   - `overflow-y: auto` — вертикальная прокрутка внутри контейнера, а не на всей странице

2. **Фиксированный блок действий**: добавлен CSS для `.booking-form__actions`:
   - `position: sticky; bottom: 12px` — прилипание к нижней части контейнера
   - `z-index: 10` — поверх остального содержимого
   - `background: var(--color-white)` — непрозрачный фон, чтобы не просвечивало содержимое
   - `border` и `border-radius` — визуальное отделение от формы

3. **Нижний отступ формы**: добавлен CSS для `#screen-booking-form .booking-form`:
   - `padding-bottom: 100px` — чтобы последние поля формы не перекрывались блоком действий при прокрутке

4. **Адаптивное расположение кнопок**: на экранах ≥ 768 px кнопки располагаются в ряд (flex-direction: row), а не в колонку, что сокращает занимаемую высоту.

## Изменённые файлы

## Использованный промпт

Промпт 5. Реализация создания бронирования (содержит требование: «мобильная вёрстка не ломается»).

## Ручная проверка

1. Открыть форму бронирования на 320 px ширины
2. Прокрутить форму до конца — кнопки видны без дополнительной прокрутки
3. Закреплены в нижней части контейнера
4. На 768+ px кнопки располагаются в ряд
5. Содержимое формы не перекрывается блоком действий
6. На всех разрешениях от 320 px до 1920 px кнопки доступны
