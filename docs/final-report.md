# Финальный отчёт — Karting Drive / Апекс

## 1. Цель работы

Разработка минимально жизнеспособного продукта (MVP) адаптивного клиентского веб-приложения для бронирования заездов в картинг-центре «Апекс». Приложение охватывает полный пользовательский путь: просмотр расписания → выбор слота → создание бронирования → просмотр и отмена своих записей — с использованием mock API для имитации backend.

## 2. Соответствие заданию летней школы

Проект выполнен в соответствии с программой летней школы:

- Реализована полноценная аналитика (бриф, выявление требований, MVP-спецификация)
- Спроектирована архитектура и модель данных
- Составлен и согласован API-контракт
- Реализованы три функции (каталог слотов, бронирование, мои бронирования)
- Написаны и выполнены тест-кейсы
- Задокументированы и исправлены баги
- Ведётся история изменений в Git

## 5. Перечень реализованных функций

1. **Каталог слотов** — загрузка расписания на 7 дней, фильтрация по дате, трассе и сложности, детальная информация о слоте, empty state, обработка ошибок API, недоступные слоты
2. **Создание бронирования** — форма с валидацией, выбор участников и экипировки, пересчёт стоимости, отправка через mock API, экран подтверждения с номером APEX-XXXXXX, обработка 409 Conflict
3. **Мои бронирования и отмена** — список бронирований с визуальными статусами, отмена с подтверждением (без alert/confirm), возврат картов после отмены, отображение отмены центром с причиной, empty state

## 6. Перечень исправленных багов

| ID | Баг | Исправление |
|----|-----|-------------|
| BUG-01 | Кнопки формы бронирования не видны без прокрутки | Sticky-блок действий, прокручиваемый контейнер, padding-bottom |
| BUG-02 | Невозможно вернуться в «Заезды» из «Мои бронирования» | Добавлен обработчик `[data-nav="slots"]`, `cursor: pointer`, выделение активного раздела |
| BUG-03 | Свободные карты не обновлялись после бронирования/отмены | Переписан `getKarts` — вычисление из confirmed-бронирований в LocalStorage |

## 7. Результат ручного тестирования

Все 34 тест-кейса пройдены (статус Passed):

- **Каталог слотов**: 11 тестов — загрузка, фильтры, empty state, детали, недоступные слоты, ошибка API, повторная загрузка
- **Создание бронирования**: 16 тестов — открытие формы, валидация, пересчёт, успешное создание, 409, 422, 503
- **Мои бронирования**: 13 тестов — отображение статусов, отмена, возврат картов, empty state, навигация
- **Интерфейс**: 7 тестов — 320/768/1920 px, горизонтальная прокрутка, cursor, ошибки JS

## 8. Использованные ИИ-инструменты и подход

- **OpenCode** — выполнение промптов, управление проектом, создание документов
- **Cursor Agent** — генерация кода (HTML, CSS, JavaScript) и технической документации
- **Qwen** (локальные и подключённые модели) — анализ требований, проверка соответствия
- **ChatGPT** — подготовка структуры промптов, верификация этапов

**Подход**: последовательное выполнение этапов через промпты от аналитики до финального оформления. Каждый промпт сохранялся в историю (`docs/prompts.md`). После каждого этапа проводилась самопроверка.

## 9. Известные ограничения

- Реальный backend не подключён — вся логика данных в mock API
- Способ идентификации клиента (TBD) — не реализована полноценная авторизация
- Онлайн-оплата и программа лояльности вынесены за рамки MVP
- Нет офлайн-режима (PWA)
- Не реализован модуль уведомлений (`notifications.js`) — указан в архитектуре, но не вошёл в текущий скоуп
