# Баг 2: Навигация между разделами

| Поле | Значение |
|------|----------|
| **ID** | BUG-02 |
| **Название** | Из раздела «Мои бронирования» нельзя было вернуться в раздел «Заезды», а кликабельные элементы не показывали cursor: pointer |

## Симптом

1. После перехода в раздел «Мои бронирования» невозможно было вернуться в раздел «Заезды» — клик по пункту навигации «Заезды» не срабатывал.
2. Кликабельные элементы навигации и кнопки не меняли курсор на pointer, что делало их неотличимыми от обычного текста.

## Серьёзность

**Высокая** — блокирует навигацию между ключевыми разделами приложения. Пользователь не мог вернуться к каталогу слотов после просмотра бронирований.

## Окружение

- Все браузеры (Chrome, Firefox, Safari, Edge)
- Все устройства

## Связанные требования

- FR-23: Навигация между экранами без перезагрузки страницы
- NFR-12: Интерактивные элементы доступны с клавиатуры (focus, Enter/Space)

## Предусловия

- Приложение загружено
- Раздел «Мои бронирования» открыт

## Шаги воспроизведения

1. Открыть приложение (отображается раздел «Заезды»)
2. Кликнуть «Мои бронирования» в навигации
3. Кликнуть «Заезды» (или логотип «Апекс») в навигации

## Ожидаемый результат

Происходит переключение на раздел «Заезды». Активный раздел визуально выделяется.

## Фактический результат

Навигация не работала в обратную сторону — после перехода в «Мои бронирования» вернуться в «Заезды» было невозможно. Кликабельные элементы не имели cursor: pointer.

## Причина

1. **Отсутствовал обработчик для кнопки «Заезды»** в навигации. В `app.js` был только обработчик для логотипа (`#nav-slots`), но не для кнопки `[data-nav="slots"]`. Функция `loadBookings` из `my-bookings.js` скрывала список слотов, а возврат не был реализован.

2. **Отсутствовал `cursor: pointer`** у элементов навигации. Кнопки навигации (`.header__nav-item`) были объявлены как `<button type="button">`, но CSS-свойство `cursor: pointer` не было указано, из-за чего браузер отображал текстовый курсор.

3. **Не работало выделение активного раздела**. При переключении между разделами класс `header__nav-item--active` не обновлялся корректно.

## Исправление

- **`js/app.js`**: добавлен обработчик для кнопки `[data-nav="slots"]`, который переключает экран на слоты и обновляет активный пункт навигации
- **`js/my-bookings.js`**: реализована функция `setActiveNav()`, которая обновляет класс `header__nav-item--active` при переключении разделов
- **`css/style.css`**: добавлен `cursor: pointer` для `.header__nav-item`

Изменения:

1. В `app.js` строках 23–29 добавлен слушатель на `[data-nav="slots"]`, вызывающий `showScreen('slots')` и `setActiveNav('slots')`
2. В `my-bookings.js` строках 24–28 реализована `setActiveNav(navId)`, управляющая классом `header__nav-item--active`
3. В `style.css` строке 99 добавлен `cursor: pointer` для `.header__nav-item`
4. Навигационные элементы используют `<button type="button">`, а не `<div>` или `<span>`, что обеспечивает семантическую правильность и клавиатурную доступность

## Изменённые файлы

- `js/app.js` — строки 23–29
- `js/my-bookings.js` — строки 24–28
- `css/style.css` — строка 99

## Использованный промпт

Промпт 6. Реализация раздела «Мои бронирования» (содержит требование навигации между экранами).

## Ручная проверка

1. Открыть приложение
2. Кликнуть «Мои бронирования» — раздел открывается, пункт выделяется
3. Кликнуть «Заезды» — раздел открывается, пункт выделяется
4. Кликнуть логотип «Апекс» — открываются «Заезды»
5. Повторить переходы 3–4 раза — навигация стабильна
6. Навести курсор на кликабельные элементы — cursor: pointer
7. Проверить консоль браузера — нет ошибок JavaScript
