# Feature-01: Каталог слотов

## Цель функции

Реализовать просмотр, фильтрацию и открытие подробностей доступных слотов заездов картинг-центра. Это первая функция проекта, основа API-взаимодействия и адаптивного интерфейса.

---

## Связанные требования

### User Stories

| User Story | Описание |
|------------|----------|
| US-01 | Просмотр расписания на 7 дней |
| US-02 | Фильтр периода |
| US-03 | Empty state при отсутствии слотов |
| US-04 | Детальная информация о слоте |

### Функциональные требования

| ID | Описание |
|----|----------|
| FR-01 | Загрузка слотов через GET /slots при старте (dateFrom=today, dateTo=today+7) |
| FR-02 | Фильтр периода (dateFrom, dateTo) с возможностью сброса |
| FR-03 | Empty state при пустом ответе API |
| FR-04 | При выборе слота отображаются: track, marshal, availableKarts, price, address, meetingPoint |

---

## Реализованное поведение

- Загрузка списка слотов при открытии страницы (по умолчанию 7 дней)
- Фильтрация по дате «от» и «до»
- Фильтрация по конфигурации трассы
- Фильтрация по уровню сложности
- Кнопка сброса всех фильтров
- Группировка слотов по дням с заголовками «Сегодня», «Завтра», дата
- Карточки слотов: время, продолжительность, трасса, сложность, маршал, свободные карты, цена, статус
- Кнопка «Подробнее» открывает блок с полной информацией о слоте
- Состояние загрузки (спиннер)
- Состояние ошибки API с сообщением и кнопкой повторной загрузки
- Empty state «Пока нет доступных заездов» при отсутствии слотов
- Недоступное состояние для отменённых/закрытых слотов (снижение прозрачности)
- Форматирование цены через Intl.NumberFormat('ru-RU')
- Форматирование даты и времени через Intl.DateTimeFormat('ru-RU')
- Адаптивная mobile-first вёрстка (1 колонка на мобильных)
- Mock API с 11 тестовыми слотами на 7 дней (разные статусы, трассы, сложности)

## Промпт 4. Реализация списка слотов

Требования к реализации:

1. Используй HTML, CSS и JavaScript без фреймворков.
2. Интерфейс должен быть адаптивным, подход mobile-first.
3. Реальный backend не используется — данные предоставляет mock API.
4. Mock API должен имитировать GET /slots и GET /slots/{slotId}.
5. Не храни расписание в LocalStorage как источник истины.

Создай не менее 8 тестовых слотов на ближайшие дни.

Каждый слот должен содержать:

- id;
- startAt;
- durationMinutes;
- status;
- trackConfiguration:
  - id;
  - name;
  - difficulty;
- marshal:
  - id;
  - name;
- availableKarts;
- maxParticipants;
- basePrice;
- equipmentOptions;
- centerAddress;
- meetingPoint;
- bookingAllowed.
