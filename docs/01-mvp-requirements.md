# Karting Drive — требования к MVP

## 1. Общая информация о проекте

| Параметр | Значение |
|----------|----------|
| Название | Karting Drive |
| Тип | Клиентское веб-приложение (учебный проект) |
| Предметная область | Бронирование заездов в картинг-центре |
| Стек | HTML, CSS, JavaScript (без фреймворков) |
| Хранение данных | LocalStorage браузера |
| Серверная часть | Отсутствует |
| Версия документа | 1.0 |
| Этап | Аналитика (MVP) |

---

## 2. Описание продукта

**Karting Drive** — одностраничное веб-приложение для картинг-центра, позволяющее пользователю самостоятельно выбрать тариф, дату и время заезда, указать количество участников, увидеть рассчитанную стоимость, собрать заказ в корзине и оформить бронирование с получением уникального номера.

Приложение работает полностью в браузере: данные о тарифах встроены в клиентский код, корзина сохраняется в LocalStorage, оформление бронирования имитируется на стороне клиента без отправки на сервер.

---

## 3. Цель проекта

Создать минимально жизнеспособный продукт (MVP), демонстрирующий полный пользовательский путь от выбора тарифа до получения номера бронирования, с использованием только клиентских технологий и без внешних зависимостей.

---

## 4. Проблема пользователя

Посетитель картинг-центра хочет быстро узнать доступные тарифы, их условия и стоимость, выбрать удобное время и забронировать заезд без звонка оператору. Без цифрового инструмента пользователь вынужден уточнять детали вручную, что замедляет принятие решения и повышает риск ошибок при расчёте стоимости и количества участников.

---

## 5. Ценность продукта

- Прозрачное отображение тарифов, ограничений и итоговой стоимости до оформления.
- Самостоятельный выбор даты, времени и количества участников с проверкой ограничений тарифа.
- Сохранение корзины между сессиями браузера (LocalStorage).
- Мгновенное получение номера бронирования без регистрации и оплаты.
- Простой и понятный интерфейс без лишних шагов.

## 7. Границы MVP

MVP охватывает **один пользовательский сценарий**: просмотр тарифов → настройка заезда → корзина → оформление бронирования → получение номера.

**Входит в MVP:**

- Статический каталог тарифов (зашит в приложение).
- Выбор даты, времени и количества участников с валидацией.
- Расчёт стоимости на клиенте.
- Корзина с редактированием и сохранением в LocalStorage.
- Форма контактных данных (имя, телефон).
- Генерация номера бронирования на клиенте.

**Не входит в MVP:**

- Сервер, API, база данных.
- Регистрация, авторизация, личный кабинет.
- Онлайн-оплата.
- Административная панель.
- Реальная проверка доступности слотов на трассе.
- Email/SMS-уведомления.
- Многоязычность.

## 11. Основные сущности предметной области

### Тариф (Tariff)

Набор условий и цены за одного участника. Атрибуты: идентификатор, название, краткое описание, полное описание, цена за участника (₽), минимальное количество участников, максимальное количество участников, минимальный возраст участника (опционально), длительность заезда (мин), доступные временные слоты.

### Заезд (Ride / Booking Item)

Конкретная позиция заказа: тариф + дата + время + количество участников + рассчитанная стоимость.

### Корзина (Cart)

Коллекция позиций заездов, хранящаяся в LocalStorage. Содержит массив позиций и, при необходимости, метаданные (дата последнего изменения).

### Позиция корзины (Cart Item)

Элемент корзины с уникальным идентификатором позиции, ссылкой на тариф, датой, временем, количеством участников и итоговой стоимостью позиции.

### Бронирование (Booking)

Подтверждённый заказ после оформления. Атрибуты: номер бронирования, имя клиента, телефон, список позиций, общая сумма, дата и время создания.

### Клиент (Customer)

Контактные данные при оформлении: имя, телефон. Не сохраняется постоянно (только в объекте бронирования на экране подтверждения).

## 12. User Stories

### US-01. Просмотр тарифов

**Как** гость,  
**я хочу** видеть список доступных тарифов с названием, кратким описанием и ценой,  
**чтобы** выбрать подходящий вариант заезда.

### US-02. Просмотр ограничений тарифа

**Как** гость,  
**я хочу** изучить полное описание и ограничения выбранного тарифа (количество участников, возраст, длительность),  
**чтобы** понять, подходит ли тариф моей группе.

### US-03. Выбор даты заезда

**Как** гость,  
**я хочу** выбрать дату заезда,  
**чтобы** забронировать заезд на удобный день.

### US-04. Запрет прошедшей даты

**Как** гость,  
**я хочу** чтобы система не позволяла выбрать дату в прошлом,  
**чтобы** не оформить невалидное бронирование.

### US-05. Выбор времени заезда

**Как** гость,  
**я хочу** выбрать время заезда из доступных слотов,  
**чтобы** приехать в удобное время.

### US-06. Выбор количества участников

**Как** гость,  
**я хочу** указать количество участников,  
**чтобы** рассчитать стоимость для всей группы.

### US-07. Проверка ограничений тарифа

**Как** гость,  
**я хочу** получать сообщение, если количество участников не соответствует ограничениям тарифа,  
**чтобы** скорректировать параметры до добавления в корзину.

### US-08. Расчёт стоимости

**Как** гость,  
**я хочу** видеть автоматически рассчитанную стоимость заезда,  
**чтобы** знать итоговую сумму до добавления в корзину.

### US-09. Добавление в корзину

**Как** гость,  
**я хочу** добавить настроенный заезд в корзину,  
**чтобы** собрать заказ из одного или нескольких заездов.

### US-10. Просмотр корзины

**Как** гость,  
**я хочу** видеть все добавленные заезды с деталями и общей суммой,  
**чтобы** проверить заказ перед оформлением.

### US-11. Изменение количества участников в корзине

**Как** гость,  
**я хочу** изменить количество участников в позиции корзины,  
**чтобы** скорректировать заказ без повторного выбора тарифа.

### US-12. Удаление позиции из корзины

**Как** гость,  
**я хочу** удалить ненужную позицию из корзины,  
**чтобы** оформить только актуальные заезды.

### US-13. Сохранение корзины в LocalStorage

**Как** гость,  
**я хочу** чтобы корзина автоматически сохранялась в браузере,  
**чтобы** не потерять выбранные заезды при случайном закрытии вкладки.

### US-14. Восстановление корзины после обновления страницы

**Как** гость,  
**я хочу** видеть ранее добавленные позиции после перезагрузки страницы,  
**чтобы** продолжить оформление с того же места.

### US-15. Ввод имени и телефона

**Как** гость,  
**я хочу** указать своё имя и телефон при оформлении,  
**чтобы** картинг-центр мог связаться со мной.

### US-16. Запрет оформления пустой корзины

**Как** гость,  
**я хочу** чтобы система не позволяла оформить пустую корзину,  
**чтобы** не создавать бессмысленное бронирование.

### US-17. Создание бронирования

**Как** гость,  
**я хочу** оформить бронирование после заполнения контактных данных,  
**чтобы** подтвердить свой заказ.

### US-18. Получение номера бронирования

**Как** гость,  
**я хочу** получить уникальный номер бронирования после оформления,  
**чтобы** предъявить его при посещении картинг-центра.

### US-19. Очистка корзины после оформления

**Как** гость,  
**я хочу** чтобы корзина очищалась после успешного оформления,  
**чтобы** начать новый заказ с чистого листа.

## 13. Критерии приёмки (Acceptance Criteria)

### US-01. Просмотр тарифов

- [ ] На главном экране отображается список всех тарифов из каталога.
- [ ] Для каждого тарифа видны: название, краткое описание, цена за участника.
- [ ] Тарифы визуально различимы (карточки или аналогичный UI-элемент).
- [ ] При клике на тариф открывается детальная информация или форма настройки заезда.

### US-02. Просмотр ограничений тарифа

- [ ] Отображается полное описание выбранного тарифа.
- [ ] Видны ограничения: мин./макс. количество участников, длительность заезда.
- [ ] При наличии — отображается минимальный возраст участника.

### US-03. Выбор даты заезда

- [ ] Доступен элемент выбора даты (input type="date" или календарь).
- [ ] Можно выбрать сегодняшнюю дату и любую будущую дату.
- [ ] Выбранная дата отображается в форме настройки заезда.

### US-04. Запрет прошедшей даты

- [ ] Даты раньше текущего дня недоступны для выбора или отклоняются при валидации.
- [ ] При попытке выбрать прошедшую дату отображается понятное сообщение об ошибке.
- [ ] Кнопка «Добавить в корзину» неактивна или заблокирована при невалидной дате.

### US-05. Выбор времени заезда

- [ ] Отображается список доступных временных слотов для выбранного тарифа.
- [ ] Можно выбрать ровно один временной слот.
- [ ] Выбранное время отображается в форме настройки заезда.

### US-06. Выбор количества участников

- [ ] Доступен элемент ввода количества участников (число, stepper или select).
- [ ] Значение по умолчанию соответствует минимуму тарифа.
- [ ] Можно изменить количество до подтверждения добавления в корзину.

### US-07. Проверка ограничений тарифа

- [ ] При количестве участников ниже минимума тарифа отображается ошибка.
- [ ] При количестве участников выше максимума тарифа отображается ошибка.
- [ ] Добавление в корзину заблокировано при нарушении ограничений.

### US-08. Расчёт стоимости

- [ ] Стоимость рассчитывается как: цена за участника × количество участников.
- [ ] Итоговая сумма обновляется при изменении количества участников.
- [ ] Сумма отображается в рублях с понятным форматированием.

### US-09. Добавление в корзину

- [ ] При нажатии «Добавить в корзину» позиция появляется в корзине.
- [ ] Позиция содержит: тариф, дату, время, количество участников, стоимость.
- [ ] Отображается подтверждение успешного добавления (уведомление или переход в корзину).

### US-10. Просмотр корзины

- [ ] Корзина отображает все добавленные позиции.
- [ ] Для каждой позиции видны: название тарифа, дата, время, участники, стоимость.
- [ ] Отображается общая сумма заказа.

### US-11. Изменение количества участников в корзине

- [ ] В корзине можно изменить количество участников для каждой позиции.
- [ ] При изменении пересчитывается стоимость позиции и общая сумма.
- [ ] При нарушении ограничений тарифа изменение отклоняется с сообщением об ошибке.

### US-12. Удаление позиции из корзины

- [ ] У каждой позиции есть кнопка удаления.
- [ ] После удаления позиция исчезает из корзины.
- [ ] Общая сумма пересчитывается.

### US-13. Сохранение корзины в LocalStorage

- [ ] При любом изменении корзины данные записываются в LocalStorage.
- [ ] Ключ хранения определён и документирован в коде.
- [ ] Сохраняются все поля, необходимые для восстановления позиций.

### US-14. Восстановление корзины после обновления страницы

- [ ] При загрузке страницы корзина читается из LocalStorage.
- [ ] Все ранее добавленные позиции отображаются корректно.
- [ ] При отсутствии или повреждённых данных инициализируется пустая корзина без ошибок.

### US-15. Ввод имени и телефона

- [ ] На экране оформления есть поля «Имя» и «Телефон».
- [ ] Поля обязательны для заполнения.
- [ ] Телефон валидируется по формату (минимум 10 цифр, допустимы пробелы, скобки, дефисы).

### US-16. Запрет оформления пустой корзины

- [ ] Кнопка «Оформить бронирование» неактивна при пустой корзине.
- [ ] При попытке оформить пустую корзину (программно) отображается сообщение об ошибке.
- [ ] Переход к оформлению недоступен без хотя бы одной позиции.

### US-17. Создание бронирования

- [ ] При успешной валидации формы создаётся объект бронирования.
- [ ] Бронирование содержит: номер, имя, телефон, позиции, общую сумму, timestamp.
- [ ] Пользователь перенаправляется на экран подтверждения.

### US-18. Получение номера бронирования

- [ ] Номер бронирования генерируется на клиенте (например, `KD-YYYYMMDD-XXXX`).
- [ ] Номер уникален в рамках сессии генерации.
- [ ] Номер отображается на экране подтверждения крупно и заметно.

### US-19. Очистка корзины после оформления

- [ ] После успешного оформления корзина очищается в памяти приложения.
- [ ] Запись корзины удаляется или перезаписывается пустым массивом в LocalStorage.
- [ ] При возврате на главный экран корзина пуста.

## 14. Основные пользовательские сценарии

### Сценарий 1. Успешное бронирование одного заезда

1. Гость открывает приложение.
2. Просматривает список тарифов.
3. Выбирает тариф «Стандарт» и изучает ограничения.
4. Выбирает дату (завтра), время (14:00), количество участников (2).
5. Видит рассчитанную стоимость: 2 × цена тарифа.
6. Нажимает «Добавить в корзину».
7. Переходит в корзину, проверяет позицию и общую сумму.
8. Нажимает «Оформить бронирование».
9. Вводит имя «Иван Петров» и телефон «+7 999 123-45-67».
10. Подтверждает оформление.
11. Видит экран с номером бронирования (например, `KD-20260704-A3F2`).
12. Корзина пуста.

### Сценарий 2. Бронирование нескольких заездов

1. Гость добавляет заезд по тарифу «Стандарт» на субботу в 12:00 (3 участника).
2. Возвращается к каталогу тарифов.
3. Добавляет заезд по тарифу «Детский» на воскресенье в 10:00 (1 участник).
4. Открывает корзину — видит 2 позиции и общую сумму.
5. Оформляет бронирование с контактными данными.
6. Получает один номер бронирования на весь заказ.

### Сценарий 3. Редактирование корзины

1. Гость добавляет заезд на 4 участников.
2. В корзине уменьшает количество до 2.
3. Стоимость позиции и общая сумма пересчитываются.
4. Удаляет позицию и добавляет новую с другими параметрами.
5. Оформляет обновлённый заказ.

### Сценарий 4. Восстановление корзины

1. Гость добавляет 2 позиции в корзину.
2. Закрывает вкладку браузера.
3. Открывает приложение снова.
4. Корзина содержит те же 2 позиции с актуальными суммами.

## 15. Альтернативные сценарии и обработка ошибок

| Ситуация | Поведение системы |
|----------|-------------------|
| Выбрана прошедшая дата | Блокировка выбора или сообщение «Нельзя выбрать прошедшую дату»; добавление в корзину недоступно |
| Количество участников ниже минимума тарифа | Сообщение: «Минимум N участников для тарифа „X"»; кнопка добавления неактивна |
| Количество участников выше максимума тарифа | Сообщение: «Максимум N участников для тарифа „X"»; кнопка добавления неактивна |
| Не выбрано время | Сообщение «Выберите время заезда»; добавление заблокировано |
| Пустая корзина при оформлении | Кнопка оформления неактивна; сообщение «Корзина пуста» |
| Пустое имя | Сообщение «Введите имя»; оформление заблокировано |
| Невалидный телефон | Сообщение «Введите корректный номер телефона»; оформление заблокировано |
| LocalStorage недоступен (приватный режим) | Корзина работает в рамках сессии; при перезагрузке данные теряются; отображается предупреждение |
| Повреждённые данные в LocalStorage | Инициализация пустой корзины; запись некорректных данных не ломает приложение |
| Изменение участников в корзине с нарушением лимита | Откат к предыдущему значению или блокировка с сообщением об ошибке |
| Двойной клик «Оформить» | Предотвращение повторного создания бронирования (debounce или блокировка кнопки) |

## 16. Функциональные требования

| ID | Требование |
|----|------------|
| **FR-01** | Приложение отображает каталог тарифов из статического источника данных (массив в JS или JSON, загружаемый на клиенте). |
| **FR-02** | Каждый тариф содержит: id, name, shortDescription, fullDescription, pricePerPerson, minParticipants, maxParticipants, durationMinutes, timeSlots[]. |
| **FR-03** | При выборе тарифа отображаются все его атрибуты, включая ограничения по участникам. |
| **FR-04** | Поле выбора даты имеет атрибут min, равный текущей дате, или эквивалентную клиентскую валидацию. |
| **FR-05** | Прошедшие даты не могут быть сохранены в позиции корзины. |
| **FR-06** | Временные слоты отображаются на основе массива timeSlots выбранного тарифа. |
| **FR-07** | Количество участников — целое число в диапазоне [minParticipants, maxParticipants] тарифа. |
| **FR-08** | Стоимость позиции = pricePerPerson × participants; общая сумма корзины = сумма стоимостей всех позиций. |
| **FR-09** | Каждая позиция корзины имеет уникальный id (UUID или timestamp-based). |
| **FR-10** | Добавление в корзину возможно только при валидных дате, времени и количестве участников. |
| **FR-11** | Корзина сериализуется в JSON и сохраняется в LocalStorage по ключу `kartingCart`. |
| **FR-12** | При загрузке приложения корзина десериализуется из LocalStorage; при ошибке парсинга — пустая корзина. |
| **FR-13** | Изменение или удаление позиции немедленно обновляет LocalStorage. |
| **FR-14** | Форма оформления содержит поля name (строка, min 2 символа) и phone (строка, ≥ 10 цифр). |
| **FR-15** | Оформление недоступно при пустой корзине или невалидной форме. |
| **FR-16** | Номер бронирования генерируется на клиенте в формате `KD-YYYYMMDD-XXXX`, где XXXX — случайный 4-символьный alphanumeric-код. |
| **FR-17** | После успешного оформления корзина очищается, LocalStorage обновляется, отображается экран подтверждения. |
| **FR-18** | Экран подтверждения показывает номер бронирования, сводку заказа и контактные данные. |
| **FR-19** | Навигация между разделами (тарифы, корзина, оформление, подтверждение) реализована без перезагрузки страницы (SPA-подход на чистом JS). |
| **FR-20** | Счётчик позиций в корзине отображается в интерфейсе (badge или текст). |

## 18. Бизнес-правила

| ID | Правило |
|----|---------|
| **BR-01** | Стоимость рассчитывается только на основе цены тарифа и количества участников; скидки и надбавки не применяются. |
| **BR-02** | Один заказ (бронирование) может содержать несколько позиций с разными тарифами, датами и временем. |
| **BR-03** | Номер бронирования присваивается одному заказу целиком, а не каждой позиции. |
| **BR-04** | Бронирование считается успешным без оплаты; оплата производится на месте в картинг-центре. |
| **BR-05** | Доступность временных слотов определяется статическим списком тарифа; реальной проверки занятости нет. |
| **BR-06** | Минимальная дата бронирования — текущий календарный день (сегодня). |
| **BR-07** | Количество участников в позиции должно соответствовать ограничениям тарифа на момент добавления и при редактировании в корзине. |
| **BR-08** | Контактные данные (имя, телефон) обязательны для оформления. |
| **BR-09** | После оформления корзина полностью очищается независимо от количества позиций. |
| **BR-10** | Тарифы и цены задаются разработчиком и не редактируются пользователем. |

## 19. Критерии готовности MVP (Definition of Done)

- [ ] Реализованы все user stories US-01 — US-19.
- [ ] Выполнены все функциональные требования FR-01 — FR-20.
- [ ] Выполнены все нефункциональные требования NFR-01 — NFR-12.
- [ ] Соблюдены все бизнес-правила BR-01 — BR-10.
- [ ] Все сценарии из раздела 14 проходят ручную проверку без ошибок.
- [ ] Все альтернативные сценарии из раздела 15 обрабатываются корректно.
- [ ] Корзина сохраняется и восстанавливается через LocalStorage.
- [ ] Номер бронирования генерируется и отображается на экране подтверждения.
- [ ] Приложение не содержит серверного кода, запросов к API и интеграций оплаты.
- [ ] Код находится в Git-репозитории с осмысленными коммитами.
- [ ] Отсутствуют критические баги, блокирующие основной пользовательский путь.
